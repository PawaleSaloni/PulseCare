# 📌 PulseCare - Requirements

## 🧠 Project Overview
PulseCare is a healthcare management system that allows patients to connect with doctors, book appointments, receive prescriptions, and manage their health records digitally.

---

## 👥 Users

- Patient
- Doctor
- Admin

---

## 🚀 Key Features

### 🔐 Authentication & Authorization
- User Registration (Patient, Doctor, Admin)
- Login functionality
- JWT-based authentication
- Role-based access control

---

### 📅 Appointment Management
- Book appointment with doctor
- View appointment history
- Cancel appointments
- Manage appointment status (Scheduled, Completed)

---

### 🩺 Doctor Management
- View list of doctors
- Search by specialization
- Manage doctor availability schedule

---

### 💊 Prescription Management (optional)
- Doctor can add prescription
- Patient can view prescription
- Store medicines and notes

---

### 🔔 Notification System (optional)
- Email notification on registration
- Appointment confirmation alerts
- (Optional) SMS notifications

---

### 💳 Payment Integration (Optional)
- Online payment for appointments
- Payment status tracking

---

### 🤖 AI Chatbot (Optional)
- Basic health-related queries
- Guidance for common symptoms

---

## 📋 Core Features (MVP)

> These features will be implemented first:

- Registration & Login
- View Doctors
- Book Appointment
- Add & View Prescription

---

## 🎯 Non-Functional Requirements

- Secure authentication (JWT)
- Fast API response
- Scalable backend (Spring Boot)
- Clean and modular code structure

---

## 🔧 Tech Stack

- Java
- Spring Boot
- Spring Data JPA
- MySQL
- Maven
